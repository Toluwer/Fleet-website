01

Quick start

Fleet supports 64-bit Windows 10 and Windows 11. Install Roblox first, then run the Fleet installer.

  1. 1
    Install Fleet

    Download FleetInstaller.exe, run it, and finish the minimal setup flow.

  2. 2
    Confirm detection

    On Instances, look for the green Roblox detected banner and Multi-instance: ready status.

  3. 3
    Launch

    Add an account or choose Signed out, select the client count, and press Launch.

No Roblox setting changes are required. Fleet detects the current player and handles path isolation automatically.

02

Accounts

Open Accounts → Add account. Fleet opens the official Roblox login page in an isolated window, so Roblox handles the username, password, 2FA, and captcha.

What Fleet stores

The resulting Roblox session cookie, encrypted with Windows DPAPI; public profile data such as avatar, display name, Robux, Premium, and presence.

What Fleet never stores

Your Roblox password, website form inputs, or plaintext session data in the renderer.

If a card says Session expired, use Sign in again. Fleet keeps the account record but never opens a login window automatically.

03

Launching clients

The Instances page supports two launch modes.

Select one or more saved accounts. Optionally paste a Place ID, Roblox game URL, or exact-server deep link. Fleet mints a separate single-use auth ticket for every selected account.

Choose how many clients to start. Each opens without a saved account and still receives a unique isolated path.

Saved sessions

After choosing accounts and an optional target, select Save current setup. A session stores account IDs, target, and optional auto-arrange—not session cookies.

04

Managing clients

Running clients show title, PID, memory, start time, origin, and responding state. Hover or right-click a row to act on it.

ActionResult
FocusRestores and raises the client window.
RestartEnds that PID and relaunches its remembered setup.
ArrangeTiles every visible Roblox window in an even grid.
EndTerminates only the selected client tree.
CleanupEnds all clients and clears leftover crash handlers.

Optional keep-alive can rejoin a crashed managed client. Cooldown and strike-out rules prevent rapid restart loops.

05

Games and Server Intelligence

Games opens with Roblox discovery categories. Search by name, hide empty experiences, keep favorites, revisit recent joins, or choose a random loaded game.

Open a game's server browser to sort by best match, ping, free space, player count, or FPS. Deep scan examines additional public server pages; occupancy, maximum ping, minimum FPS, and free-slot filters can be combined.

Join best always uses the top server after the active filters and sort order—not the unfiltered list.

06

People and server roster

Search Roblox users or browse friends merged across saved accounts. Presence refreshes patch only the affected card instead of redrawing the page.

The Server subtab can inspect player objects from a currently running local Roblox process. It shows detected users, avatars, IDs, team, and health when the current Roblox memory layout exposes them. Reads are bounded by time and count and run in a worker.

07

Settings

  • Theme: System, Light, or Dark, including native Windows controls.
  • Roblox location: Automatic registry/filesystem detection or a manual executable path.
  • Refresh interval: Frequency for the running-client snapshot.
  • Delay between launches: Startup spacing for multi-launches.
  • Confirmation: Optional prompts before End all and Cleanup.
  • Updates: Installed NSIS builds can check a reachable GitHub release feed.
08

Architecture

Fleet separates the node-free renderer, a narrow preload bridge, and the privileged Electron main process. Context isolation is enabled; the renderer reaches operating-system functions only through explicit IPC channels.

Process model
renderer/  →  preload bridge  →  main/
pure UI       explicit IPC       Win32, auth, storage

instance-N/RobloxPlayerBeta.exe
    └── junction → installed Roblox version

The native layer uses koffi for Toolhelp32 process enumeration, memory statistics, window focus/arrangement, and the handle operations needed for the shared singleton guard.

09

Troubleshooting

Roblox was not found

Install Roblox, choose Re-detect, or switch to Manual path and select RobloxPlayerBeta.exe under the latest Roblox Versions folder.

A newly opened client closes

Increase Delay between launches and confirm the footer reads Multi-instance: ready. Diagnostics will show whether the native helper loaded.

Multi-instance is unavailable

Reinstall Fleet so the packaged koffi native module is restored. Single-client launch and process management remain available.

Focus does not raise a window

Windows can refuse foreground changes while another app owns focus. Fleet still restores and raises the window; use the taskbar button if Windows blocks activation.

10

Build from source

Fleet requires Node.js 18 or newer and runs on Windows.

PowerShell
npm install
npm start

# Core verification
npm run selftest
npm run test:ui

# Portable folder and official installer
npm run build
npm run dist

The installer is written to dist\FleetInstaller.exe. Current public builds are unsigned unless a Windows code-signing certificate is configured.
