What it does

The multi-client loop,
without the juggling.

Fleet covers the work around Roblox that gets messy when one client becomes several.

01

Launch separately

Start signed-in or signed-out clients through lightweight path isolation while leaving the installed Roblox player untouched.

02

Manage every window

See live processes, focus or arrange windows, restart a session, end one client, or clean up the whole setup.

03

Choose the server

Search experiences, deep-scan joinable servers, and sort by ping, space, players, FPS, or a combined quality score.

04

Find people

Search public profiles, merge friends across accounts, inspect a local server roster, and join when Roblox exposes a presence.

05

Save the setup

Store account groups, game or server targets, and arrangement preferences as reusable local sessions.

06

See what happened

Use local playtime stats, launch history, and live diagnostics to understand every session without a cloud dashboard.

Inside Fleet

Small shell.
Serious local control.

Version 1.5.2 moved Fleet to Tauri 2 for a lighter native shell while keeping the renderer and Windows service layer explicit.

WebView rendererVanilla HTML, CSS, and JavaScript
→
Tauri commandsRust shell and narrow bridge
→
Local service hostWin32, accounts, launch, and storage

Path isolation

Each client launches through its own directory junction, creating a unique executable path without copying Roblox files.

Targeted guard

Fleet closes only the shared singleton objects that block later launches and leaves each client's path-derived mutex intact.

Local data

Settings, account records, history, logs, and saved sessions stay in Fleet's application-data directory on this PC.
